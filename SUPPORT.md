# Поддержка

## Как Получить Помощь

Используйте тот канал, который лучше всего подходит под ваш запрос:

- bug report: откройте GitHub issue с шагами воспроизведения
- feature request: откройте GitHub issue и опишите сценарий использования
- предложение по обновлению контента: откройте issue или pull request
- проблема безопасности: не публикуйте её публично, используйте [SECURITY.md](./SECURITY.md)

Владелец репозитория:

- <https://github.com/minkinad>

## Что Проверить Перед Созданием Issue

Проверьте:

- [README.md](./README.md)
- существующие открытые issues
- существующие pull requests
- воспроизводится ли проблема на актуальном `main`

## Что Стоит Указать В Сообщении

Для технических проблем желательно приложить:

- ожидаемое поведение
- фактическое поведение
- шаги воспроизведения
- скриншоты, если это уместно
- затронутую страницу, трек или маршрут
- вывод локальных команд, если проблема связана со сборкой, lint или validation

Полезные команды:

```bash
npm run prepare:content
npm run validate:content
npm run lint
npm run typecheck
npm run build
```

## В Чём Мы Можем Помочь Лучше Всего

Сопровождающий проекта лучше всего может помочь с:

- структурой репозитория
- процессом внесения изменений
- проблемами рендера документации
- синхронизацией и валидацией контента
- генерацией поискового индекса
- CI и деплой-пайплайном

Поддержка не означает персональный консалтинг, написание полного учебного курса под заказ или мгновенный turnaround по ревью.

## Как Ускорить Ответ

- формулируйте проблему узко и конкретно
- указывайте точные пути к файлам
- прикладывайте шаги воспроизведения
- разделяйте независимые проблемы на отдельные issues
- используйте draft PR, если у вас уже есть возможное исправление
