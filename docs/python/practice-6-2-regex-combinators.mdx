## Теория

Практика продолжает функциональный подход и собирает парсер из маленьких функций-комбинаторов:

- `sym` и `range_of` читают символы;
- `seq` задаёт последовательность;
- `alt` задаёт альтернативы;
- `group` выделяет захваченные части разбора.

## Код из решения

```python title="pract6_2.ipynb (parser combinators)"
def sym(c):
    def parse(text, pos):
        if pos < len(text) and text[pos] == c:
            return pos + 1
        return None
    return parse

def range_of(start, end):
    def parse(text, pos):
        if pos < len(text) and start <= text[pos] <= end:
            return pos + 1
        return None
    return parse

def alt(*parsers):
    def parse(text, pos):
        for p in parsers:
            new_pos = p(text, pos)
            if new_pos is not None:
                return new_pos
        return None
    return parse

def seq(*parsers):
    def parse(text, pos):
        for p in parsers:
            pos = p(text, pos)
            if pos is None:
                return None
        return pos
    return parse

digit = range_of('0', '9')
hex_digit = alt(digit, range_of('a', 'f'), range_of('A', 'F'))
hex_color = seq(sym('#'), hex_digit, hex_digit, hex_digit, hex_digit, hex_digit, hex_digit)

print(hex_color('#ffaa43') is not None)
```

## Разбор

Комбинаторы позволяют прозрачно читать правила разбора и переиспользовать их как обычные функции. Это хорошая база для собственных DSL и простых интерпретаторов.

Исходник практики: `pr_Python/pr_Python/pract6_2.ipynb`.
